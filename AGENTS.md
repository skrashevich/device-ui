## Cursor Cloud specific instructions

### Project Overview

This is the **Meshtastic device-ui library** — a C++ UI library built with LVGL v9.3 for Meshtastic LoRa mesh networking devices. It targets TFT/OLED displays on embedded platforms (ESP32, nRF52, Raspberry Pi) and provides a native Linux/X11 simulation for development.

This repository produces a **static library** (`libDeviceUI.a`) and unit tests — there is no standalone executable application. The "hello world" for this project is building the library and running the test suite.

### Build (CMake + GCC)

The default system compiler may be Clang which fails to link due to `libstdc++` path issues. Always specify GCC explicitly:

```bash
mkdir -p build && cd build
cmake -DCMAKE_C_COMPILER=/usr/bin/gcc -DCMAKE_CXX_COMPILER=/usr/bin/g++ \
      -DLV_BUILD_CONF_DIR=/workspace/include ..
make -j$(nproc)
```

Key gotchas:

- `LV_BUILD_CONF_DIR` must be an **absolute path** to `/workspace/include` (not relative `"include"`). The LVGL cmake cache variable resolution is relative to the build directory otherwise.
- CMake `FetchContent` downloads all dependencies (Portduino, LovyanGFX, LVGL, nanopb, protobufs, doctest) from GitHub on first configure. Subsequent builds reuse the cached `_deps/` folder.
- The nanopb protobuf generator requires `python3` with the `protobuf` and `grpcio-tools` packages installed.

### Testing

```bash
cd build && ctest --output-on-failure
# Or directly:
./build/bin/tests
```

Tests use the doctest framework. 7 test cases, 1039 assertions (MapPanel scroll + GeoPoint).

### Linting

Trunk.io is used for linting (auto-initialized with `trunk init -y` on first use):

```bash
trunk check              # check all modified files
trunk check <file>       # check specific file
trunk fmt <file>         # auto-format
```

The `.trunk/trunk.yaml` config is auto-generated by `trunk init` and not committed upstream. Future agents should run `trunk init -y` if the `.trunk/` directory is missing.

### System Dependencies (required beyond what the update script installs)

These are installed in the VM snapshot (not in the update script since they are system packages):

- `libinput-dev`, `libxkbcommon-dev` — LVGL input driver support
- `libi2c-dev` — Portduino I2C support
- `libx11-dev` — X11 display driver (pre-installed)
- `protobuf-compiler` — protoc for nanopb generator
- `xvfb` — virtual X11 framebuffer for headless GUI testing

### X11 / Display

For any X11-dependent operations (e.g., future GUI simulation testing), start Xvfb:

```bash
Xvfb :99 -screen 0 1024x768x24 &
export DISPLAY=:99
```

### Node.js / lv_i18n

The `lv_i18n` tool is installed globally via npm (per `.devcontainer/devcontainer.json` postCreateCommand). It is used for locale/translation management.
